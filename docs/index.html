Why use it

Built to stay out of your way.

Designed around three goals: accurate detection, safe merges, and no surprises on re-runs.

Automatic detection

Recognises 50+ stacks — Node, Python, Java, Docker, Rust, Go, .NET, AI tooling — by scanning files and extensions.

Idempotent merge

Running it again produces the same file. Your hand-written rules live outside the auto-generated markers.

Semantic dedupe

Collapses node_modules, node_modules/, **/node_modules/ — keeping negations intact.

Robust fallbacks

GitHub templates first, then gitignore.io, then local bundles. Timeouts and retries for flaky networks.

Monorepo-aware

Opt-in nested scanning via --depth detects apps/*, services/* and workspaces.

Zero config

Run npx smart-gitignore and done. No config file, no flags required.

Usage

One command.

Runs anywhere with Node.js 18+. No config file, no setup step.

Recommended — no install required

$ npx smart-gitignore

Or install globally

$ npm install -g smart-gitignore

Available options

--dir <path> Directory to scan. Defaults to CWD.
--depth <n> Scanner depth. Use ≥2 for monorepos.
--force Overwrite instead of merging.
--verbose Show detection and fetch steps.
--timeout <ms> Per-request HTTP timeout.
--retries <n> HTTP retry attempts on failure.
